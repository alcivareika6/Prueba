En el estándar HTML5 se consideró la idea de que se usaran varios <h1> independientes por página basados en el llamado “outline algorithm” de secciones.

Ese algoritmo sí se propuso entre 2008 y 2010, y sobre esa base se popularizó la idea de “un <h1> por section o article”. Se consideraba válido (e incluso correcto) tener múltiples <h1> en una misma página, cada uno como encabezado principal de su sección.
