En el estándar HTML5 se consideró la idea de que se usaran varios <h1> independientes por página basados en el llamado “outline algorithm” de secciones.

Ese algoritmo sí se propuso entre 2008 y 2010, y sobre esa base se popularizó la idea de “un <h1> por section o article”. Se consideraba válido (e incluso correcto) tener múltiples <h1> en una misma página, cada uno como encabezado principal de su sección.

El estándar decía: "No te preocupes por los niveles (h2, h3), usa siempre h1 dentro de las secciones". Esto era genial para la portabilidad: podías coger un <article> de una página, pegarlo en otra, y no tenías que reescribir los h2 a h3. Si además el desarrollador utilizaba CSS BEM, que por esas fechas estaba eclosionando, “outline algorithm” aportaba aún más poder para la reutilización del códico.

La realidad

Lamentablemente, ningún navegador principal (Chrome, Firefox, Safari) ni los lectores de pantalla (como JAWS o NVDA) implementaron este algoritmo correctamente. Esto afecta a la accesibilidad del código.

Debido a este fracaso en la implementación, el estándar se ha actualizado. Nota del estándar actual:

"No hay implementaciones del algoritmo de esquema... Por lo tanto, se aconseja a los autores usar títulos de rango correcto (h1-h6) para transmitir la estructura del documento."

Siendo que la comunidad no ha aceptado “outline algorithm”, el código basado en CSS BEM es un poco menos portable.
